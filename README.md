# Walmart Sales Performance Analysis using SQL

## Project Overview
This project analyzes Walmart sales data using MySQL to uncover business insights related to sales performance, customer behavior, and product trends.

The analysis was performed using advanced SQL techniques including aggregations, window functions, subqueries, and data transformations.

## Tools Used
- MySQL Workbench
- SQL
- Excel
- PowerPoint

## Dataset
The dataset contains transaction-level data including:
- Branch
- City
- Customer Type
- Gender
- Product Line
- Payment Method
- Sales Revenue
- Date of Purchase

## Key Analysis Tasks

### 1. Top Branch by Sales Growth
Identified which Walmart branch showed the highest sales growth over time.

### 2. Most Profitable Product Line
Determined which product lines generated the highest profit for each branch.

### 3. Customer Segmentation
Segmented customers into High, Medium, and Low spenders based on purchase amounts.

### 4. Sales Anomaly Detection
Detected unusually high or low transactions compared to the product line average.

### 5. Most Popular Payment Method
Analyzed preferred payment methods across cities.

### 6. Monthly Sales Distribution by Gender
Examined sales patterns between male and female customers.

### 7. Best Product Line by Customer Type
Compared product preferences between Member and Normal customers.

### 8. Repeat Customer Analysis
Identified customers making repeat purchases within 30 days.

### 9. Top Customers by Revenue
Found the top 5 customers generating the highest sales.

### 10. Weekly Sales Trends
Analyzed which days of the week generate the highest sales.

## Key Insights
- Certain branches showed stronger sales growth trends.
- Specific product lines contributed the highest profits.
- Weekend sales were significantly higher than weekdays.
- Payment preferences varied across cities.
- Repeat customers contributed a significant portion of revenue.
